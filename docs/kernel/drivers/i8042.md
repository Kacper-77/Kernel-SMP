# PS/2 Controller (i8042) Driver

## Overview
The i8042 module provides a driver for the Intel 8042 Keyboard Controller (KBC). It serves as the primary interface for legacy PS/2 input devices, specifically the keyboard. The driver handles controller initialization, command/data transport, and status monitoring via I/O ports.

## Design Decisions

1. SMP Safety and Synchronization
The most critical design aspect of this driver is its SMP-safe implementation. Since the i8042 controller is a shared global resource, concurrent access from multiple cores could lead to corrupted command sequences or interleaved data.
- Spinlock Protection: Every I/O operation (read_data, write_data, write_command) is protected by a global spinlock (i8042_lock_).
- Interrupt State Management: The driver utilizes spin_irq_save() and spin_irq_restore() to disable local interrupts during the lock acquisition. This prevents potential deadlocks if an interrupt handler attempts to access the controller while the core already holds the lock.

2. Polling and Busy-Waiting
To ensure the controller is ready for communication, the driver implements polling of the Status Register:
- Wait for Write: Monitors the PS2_STATUS_INPUT_FULL bit to ensure the input buffer is empty before sending commands or data.
- Wait for Read: Monitors the PS2_STATUS_OUTPUT_FULL bit to ensure data is available before reading.
- CPU Backoff: Uses the 'pause' instruction within polling loops to reduce power consumption and improve spin-loop performance on modern x86 processors.

## Technical Details

Port Mapping:
- Port 0x60: Data Port (Read/Write)
- Port 0x64: Status Register (Read-only)
- Port 0x64: Command Register (Write-only)

Initialization Sequence:
The i8042_init() function follows a rigorous setup procedure:
1. Device Disabling: Both PS/2 ports (P1 and P2) are disabled to prevent interference during configuration.
2. Buffer Flushing: Ensures no stale data remains in the controller's output buffer.
3. Controller Self-Test: Sends command 0xAA and expects 0x55 to verify hardware integrity.
4. Configuration Mapping: Reads the Controller Configuration Byte, enables IRQ 1 (Keyboard), and disables translation to maintain raw scancode integrity.
5. Port Activation: Enables the first PS/2 port and sends the 0xF4 (Enable Scanning) command to the keyboard device.

## Future Improvements
- PS/2 Mouse Support: Extend the driver to initialize and handle the second PS/2 port (P2) and IRQ 12.
- Advanced Error Handling: Incorporate checks for PS2_STATUS_TIMEOUT and PS2_STATUS_PARITY flags.
- Non-blocking Writes: Implement an interrupt-driven output buffer management system to eliminate busy-waiting in performance-critical sections.
